# 📋 任务拆解模板

## 功能开发任务拆解

```markdown
## 🎯 任务：[功能名称]

### 验收标准
- [ ] 功能描述：[一句话描述]
- [ ] 输入：[接受的输入类型/格式]
- [ ] 输出：[预期输出结果]
- [ ] 性能：[响应时间/吞吐量要求]
- [ ] 安全：[安全相关要求]

### 任务分解

#### Phase 1: 设计与准备 (预计：__分钟)
- [ ] 分析需求，明确边界条件
- [ ] 设计接口/函数签名
- [ ] 确定依赖关系
- [ ] 准备测试数据

#### Phase 2: 核心实现 (预计：__分钟)
- [ ] 实现主流程逻辑
- [ ] 实现输入验证
- [ ] 实现错误处理
- [ ] 添加关键注释

#### Phase 3: 测试验证 (预计：__分钟)
- [ ] 编写单元测试（核心路径）
- [ ] 编写单元测试（边界条件）
- [ ] 编写单元测试（错误处理）
- [ ] 运行测试并验证覆盖率

#### Phase 4: 优化与文档 (预计：__分钟)
- [ ] 代码重构优化
- [ ] 添加类型定义
- [ ] 更新相关文档
- [ ] 准备提交信息

### 风险与依赖
| 风险项 | 可能性 | 影响 | 缓解措施 |
|--------|--------|------|----------|
| [风险 1] | 低/中/高 | 高/中/低 | [应对措施] |
| [依赖项] | - | - | [前置任务] |

### 完成定义 (DoD)
- [ ] 代码通过所有测试
- [ ] 类型检查通过
- [ ] 代码规范检查通过
- [ ] 安全审查通过
- [ ] 文档已更新
```

## Bug 修复任务拆解

```markdown
## 🐛 Bug 修复：[Bug 描述]

### Bug 信息
- **复现步骤**: [1. 2. 3. ...]
- **预期行为**: [应该发生什么]
- **实际行为**: [实际发生了什么]
- **影响范围**: [哪些功能/用户受影响]
- **优先级**: 🔴 Critical / 🟡 High / 🔵 Medium / ⚪ Low

### 根因分析
```
[分析导致 bug 的根本原因]
```

### 修复方案
- [ ] 方案 A: [描述] - 风险：低/中/高
- [ ] 方案 B: [描述] - 风险：低/中/高

**选择方案**: [选择的方案及理由]

### 修复步骤
- [ ] 编写复现测试（失败）
- [ ] 实施修复代码
- [ ] 验证测试通过
- [ ] 回归测试（确保无新 bug）
- [ ] 更新相关文档

### 验证清单
- [ ] 原 bug 已修复
- [ ] 无新 bug 引入
- [ ] 性能无明显下降
- [ ] 相关测试已更新
```
